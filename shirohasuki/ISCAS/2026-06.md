### 本月工作总结

本月主要围绕 trace 框架和 buckyball 后端移植展开，目标是让 buddy-compiler 具备更完整的模型调试、性能分析和 buckyball 后端支持能力。主要内容如下：

1. trace框架
- 支持 cycle Trace，用于模型运行过程中的性能 profiling
- 支持 Tensor Trace，用于模型中间结果导出和精度 debug
- 支持多级 trace，可以按 node 和 dialect 选择 trace 
- 支持为不同模型配置不同 trace 点，便于定位不同模型的精度或性能问题
- 支持 trace 数据与后续可视化、profiling 和对比分析流程衔接

2. buckyball移植

- 添加 Tile Dialect 和对应pass，用于在 linalg 和具体 DSA 后端之间提供统一中间抽象
- 添加 buckyball external dialect 支持，用于在 buddy-mlir 中对接外部 buckyball 后端
- llvm 后端支持 buckyball 相关 RISC-V vendor extension
- 添加 buckyball 基础算子和 kernel 级测试，用于验证 buckyball 后端基础编译链路

3. bug修复
- 修复大常量处理相关问题，避免模型编译阶段因常量规模过大失败

### 本月合并的PR

[Add buckyball as a new backend, and add trace tools](https://github.com/buddy-compiler/buddy-mlir/pull/814) 

该 PR 主要在 buddy-mlir 中增加 buckyball 后端接入所需的编译器基础设施，并添加 trace 调试框架，共包含60个commit。

1. Trace实现的修改包括：
- 添加 Trace Dialect，用于在 IR 中表示 tensor trace 的插桩点和 trace 操作
- 添加 TraceInsert pass，用于在 linalg 和 buckyball 等不同层级的 IR 中插入 trace
- 添加 `-extend-trace-to-linalg` pass 的示例，用于将外层 trace 扩展到匹配的 `linalg.matmul` 等 linalg op 上
- 添加 frontend trace transform，使 Python frontend 可以在图层面声明 trace 信息并传递到后续编译流程
- 添加基于 perfetto 的 trace 数据分析
- 添加 trace 相关测试

2. buckyball接入的修改包括：
- 添加 Tile Dialect，用于在 linalg 和具体 DSA 后端之间提供统一的 tile-level 抽象
- 添加 LowerLinalgToTile 转换，为 buckyball 后端复用统一的中端 lowering 路径
- 添加 external dialect 支持，用于在 buddy-mlir 中对接外部 buckyball dialect
- 添加 buckyball 基础示例和测试，包括 quant、dequant、im2col、relu、systolic、tile matmul、transpose 等
- 添加 buckyball 相关构建和测试配置，使新增后端可以进入 buddy-mlir 的 CMake、lit 和示例体系


[Add cycle and tensor tracing capabilities for all models](https://github.com/buddy-compiler/buddy-examples/pull/11)

该 PR 基于前面在 buddy-mlir 中构建的 trace 框架，为 buddy-examples 中的端到端模型补充 cycle trace 和 tensor trace 支持，并同步补齐 buckyball 后端通路。


[[RISCV] Add Buckyball RISC-V vendor extensions](https://github.com/RuyiAI-Stack/llvm-project/pull/20)

该 PR 将 buckyball 相关指令整理为 RISC-V vendor extension，添加至维护的 llvm-project。该 PR 增加了三类扩展：

1. `+xbuckyballbase`：用于 buckyball 共享基础指令，主要服务于 memory domain 和 frontend 控制等基础能力。
2. `+xbbpebble`：用于 CNN 方向的 Pebble 指令拓展。
3. `+xbbmeteor`：用于 LLM 方向的 Meteor 指令拓展。

该 PR 与 buddy-mlir 中的 buckyball 接入工作相互配合，两部分结合后，完成了buckyball后端接入的闭环。
